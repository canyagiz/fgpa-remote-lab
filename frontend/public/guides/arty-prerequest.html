1What a LUT actually is

Everything the FPGA does — including this entire neural network — ultimately runs on these tiny memory blocks.

A LUT (Look-Up Table) is a tiny block of SRAM. Its inputs act as an address; the output is whatever bit is stored at that address. A 6-input LUT stores a 64-bit truth table, so any function of up to 6 variables can be implemented just by writing the right bits into it — this is most of what "programming" an FPGA means.

The Arty Z7 has 53,200 of these LUTs, all operating in parallel. That parallelism — not a faster clock — is why an FPGA can run CNN inference so efficiently. BRAM and DSP48 blocks (used later for weight storage and multiply-accumulate) are separate dedicated blocks, not built out of LUTs.

2Training vs. Inference

The board you'll program never learns anything — that already happened, on a PC, before the lab existed.

Training — happens once, in software

  • Forward pass and backward pass
  • Weights updated over many epochs
  • Runs on a developer's PC (TensorFlow/PyTorch)
  • Produces one fixed set of weight values

Inference — what runs on your FPGA

  • Forward pass only
  • Weights are fixed, loaded once into BRAM
  • One image in → one predicted digit out
  • Exactly the arithmetic FPGA hardware is good at

3The MNIST dataset

28×28 grayscale handwritten digits, 0–9 — the same images your input navigator in the lab will step through.

Each image is a single-channel 28×28 pixel grid (784 values, 0–255). It's the standard benchmark for a first CNN because it's small enough to run comfortably in FPGA fabric, yet realistic enough that getting 98%+ accuracy is a genuine result.

5One weight file, many student designs

The provided .coe file contains only numbers — never an architecture.

Fixed for everyone

  • Layer count and order
  • Filter counts and kernel sizes
  • The actual weight values

Yours to design

  • Parallel vs. serial multiply-accumulate
  • Pipeline depth
  • Internal fixed-point bit-widths
  • BRAM access patterns

Because the architecture is fixed, two completely different hardware designs can load the same .coe file and both produce the correct classification — just at different speeds and resource costs. The provided testbench checks only input/output correctness, never internal structure.

6What "weight quantization" means (Lab 2 preview)

Same trained network, same architecture — only the number format representing each weight changes.

scale = 127 / max(abs(weights))  →  int8_value = round(weight × scale)

FormatBits/weightTypical effect
FP3232Reference accuracy, most expensive in FPGA fabric
INT88~4× less BRAM, DSP48-friendly, usually small accuracy cost
INT44Further resource savings, accuracy starts to drop
INT22Cheapest, but accuracy can degrade substantially
